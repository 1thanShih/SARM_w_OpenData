Route C · 手動標記

lerobot-annotate
→ SARM 完整教學

使用 LeRobot 官方 Web UI 對 charging bimanual 資料集進行手動 subtask 標記， 再轉換成 SARM dense_only 訓練格式。

🤖 Dataset：charging_bimanual_merged（284 episodes） 📷 Camera：observation.images.top ⚙️ Mode：dense_only

0總覽與背景

SARM（Stage-Aware Reward Modeling）需要每個 episode 的 subtask 邊界資訊來計算進度標籤（progress labels）。 預設用 Qwen VLM 自動標記，但若 VLM 效果不佳，改為手動標記是最可靠的替代方案。

整體流程

已完成Merge Dataset
→
本教學手動標記
→
本教學格式轉換
→
下一步SARM 訓練
ℹ️

不需要對全部 284 集做手動標記。建議先從每個 source repo 抽 5 集（共 30 集）標記， 驗證品質後再決定是否擴大。

!格式差異說明（必讀）

lerobot-annotate 工具的輸出格式與 SARM 要求的格式不同， 需要一個轉換步驟。了解差異可以避免後面出錯。

項目 lerobot-annotate 輸出 SARM 要求
資料位置 data/chunk-*/file-*.parquet episodes/chunk-*/episode_*.parquet
粒度 每 frame 一列 每 episode 一列
subtask 欄位 subtask_index（int，每 frame） dense_subtask_names（list[str]）
dense_subtask_start_frames（list[int]）
dense_subtask_end_frames（list[int]）
額外 metadata meta/subtasks.parquet meta/temporal_proportions_dense.json

Step 5 的轉換腳本會自動處理這個差異，你不需要手動操作。

*本任務的 Subtasks

Charging Bimanual 任務分成 4 個 subtask，標記時請使用以下完全一致的名稱：

#Subtask 名稱（請原文複製）動作說明
1Pick up the phone雙手從起始位置抬起並抓取手機
2Flip the phone sideways將手機翻轉到橫向（充電孔朝外）
3Pick up the charging cable and plug it into the phone抓起充電線並插入手機
4Turn on the power of the extension cord按下延長線開關通電
⚠️

subtask 名稱必須在所有 episode 保持一致（大小寫也算）。轉換腳本依靠名稱來建立 temporal_proportions_dense.json，名稱不一致會導致比例計算出錯。

3標記 Episodes

每一集的標記流程如下：

  1. 選擇 episode

    從左側列表點選一集，影片會在中間區域播放。

  2. 先完整看一遍影片

    了解這集機器手臂的動作節奏，找出每個 subtask 的大概轉換點。

  3. 新增 subtask segment

    點 Add Subtask Segment，拖動時間軸標記開始/結束時間， 在 Label 欄位輸入 subtask 名稱（從上方的 Subtasks 表格複製）。
    依序標記 4 個 subtask。

  4. 確認標記正確

    在 timeline 上確認 4 個 segment 連續覆蓋整個影片（沒有重疊也沒有空隙）， 播放各段預覽確認動作吻合。

  5. Save 這集

    點 Save Episode，標記會寫入 meta/lerobot_annotations.json（可 resume）。

  6. 重複以上步驟

    建議先標每個 source repo 各 5 集（共 30 集），再全部做。

失敗的 episode（機器手臂中途掉落或任務未完成）也要標記。 將失敗點之後的 subtask 標記為「沒有完成」的起點到影片結尾即可， SARM 會透過 progress signal 自動降低這些 frames 的權重。

5轉換成 SARM 格式

執行下方的轉換腳本，會自動把 lerobot-annotate 的 per-frame 格式 轉成 SARM 要求的 per-episode 格式，並計算 temporal_proportions_dense.json。

準備轉換腳本

把以下腳本存成 convert_annotate_to_sarm.py：

convert_annotate_to_sarm.py
import json
import pandas as pd
import numpy as np
from pathlib import Path
import shutil

# ── 設定 ─────────────────────────────────────────────
EXPORT_DIR  = Path("./lerobot_annotate_export")  # lerobot-annotate export 路徑
OUTPUT_DIR  = Path("./sarm_ready_dataset")        # 轉換後輸出路徑
FPS         = 30
# ─────────────────────────────────────────────────────

# 1. 讀 subtask 名稱對照表
subtasks_df = pd.read_parquet(EXPORT_DIR / "meta/subtasks.parquet")
idx_to_name = dict(zip(subtasks_df.index, subtasks_df["label"]))
print(f"✓ 讀到 {len(idx_to_name)} 個 subtask 名稱: {list(idx_to_name.values())}")

# 2. 掃描所有 data parquet，收集每 episode 的 subtask 邊界
per_episode = {}  # episode_index -> {names, starts, ends}

for fpath in sorted((EXPORT_DIR / "data").rglob("*.parquet")):
    df = pd.read_parquet(fpath)
    if "subtask_index" not in df.columns:
        continue
    for ep_idx, grp in df.groupby("episode_index"):
        sub     = grp["subtask_index"].values
        names   = []
        starts  = []
        ends    = []
        frame_offset = int(grp["frame_index"].iloc[0]) if "frame_index" in grp.columns else 0
        i = 0
        while i < len(sub):
            if sub[i] == -1:
                i += 1
                continue
            label = sub[i]
            j     = i
            while j < len(sub) and sub[j] == label:
                j += 1
            names.append(idx_to_name[label])
            starts.append(int(i + frame_offset))
            ends.append(int(j - 1 + frame_offset))
            i = j
        per_episode[ep_idx] = {
            "dense_subtask_names":        names,
            "dense_subtask_start_frames":  starts,
            "dense_subtask_end_frames":    ends,
        }

print(f"✓ 解析 {len(per_episode)} 個 episode 的標記")

# 3. 計算 temporal_proportions_dense（各 subtask 平均佔比）
lengths_by_name = {}
total_frames    = []

for r in per_episode.values():
    if not r["dense_subtask_end_frames"]:
        continue
    ep_len = r["dense_subtask_end_frames"][-1] + 1
    total_frames.append(ep_len)
    for name, s, e in zip(
        r["dense_subtask_names"],
        r["dense_subtask_start_frames"],
        r["dense_subtask_end_frames"]
    ):
        lengths_by_name.setdefault(name, []).append(e - s + 1)

mean_total = np.mean(total_frames) if total_frames else 1
raw = {
    name: float(np.mean(lens) / mean_total)
    for name, lens in lengths_by_name.items()
}
total = sum(raw.values())
proportions = {k: v / total for k, v in raw.items()}
print(f"✓ temporal_proportions: {proportions}")

# 4. 複製 export，再寫回 episodes parquet
if OUTPUT_DIR.exists():
    shutil.rmtree(OUTPUT_DIR)
shutil.copytree(EXPORT_DIR, OUTPUT_DIR)

# 找 episodes parquet（v3.0 格式）
ep_parquet_dir = OUTPUT_DIR / "episodes"
if not ep_parquet_dir.exists():
    # fallback：建立空的 episodes 目錄
    ep_parquet_dir.mkdir(parents=True)

for ep_idx, r in per_episode.items():
    chunk   = ep_idx // 1000
    ep_file = ep_parquet_dir / f"chunk-{chunk:03d}/episode_{ep_idx:06d}.parquet"
    ep_file.parent.mkdir(parents=True, exist_ok=True)

    if ep_file.exists():
        df = pd.read_parquet(ep_file)
    else:
        df = pd.DataFrame({"episode_index": [ep_idx]})

    df["dense_subtask_names"]       = [r["dense_subtask_names"]]
    df["dense_subtask_start_frames"] = [r["dense_subtask_start_frames"]]
    df["dense_subtask_end_frames"]   = [r["dense_subtask_end_frames"]]
    df.to_parquet(ep_file, index=False)

# 5. 寫 temporal_proportions_dense.json
with open(OUTPUT_DIR / "meta/temporal_proportions_dense.json", "w") as f:
    json.dump(proportions, f, indent=2, ensure_ascii=False)

print(f"\n✅ 轉換完成！輸出目錄: {OUTPUT_DIR.resolve()}")
print(f"   標記 episodes: {len(per_episode)}")

?常見問題

❌ lerobot-annotate 開啟後看不到影片

工具需要 ffmpeg。確認系統有安裝：

bash
# Ubuntu / Colab
sudo apt-get install ffmpeg

# Mac
brew install ffmpeg

❌ dataset 格式不相容（找不到 episodes 目錄）

lerobot-annotate 可能只認舊版格式（data/chunk-*/file-*.parquet）。 如果工具無法讀取 v3.0 格式，改用本地 snapshot_download 下載後連接本地路徑。

❌ 轉換後 temporal_proportions 數值不合理（某 subtask 比例 > 0.7）

表示部分 episode 的某個 subtask 佔了大部分時間，可能是標記位置不準確。 回到工具修正那幾集，或者先排除明顯異常的集重新計算。

❌ 訓練時出現 KeyError: dense_subtask_start_frames

SARM 從 episodes/*.parquet 讀取標記欄位，但你的 push 可能沒有包含這個目錄。 確認執行：

bash
# 確認 episodes 目錄被 push 上去
huggingface-cli repo files 你的帳號/charging_bimanual_annotated --repo-type dataset | grep episodes

★標記技巧

情況 建議做法
兩個 subtask 之間有短暫停頓 停頓包進前一個 subtask 的結尾（不要讓 segment 有空隙）
動作失敗後重試 把整段重試都包在同一個 subtask，不要拆開
影片開頭機器手臂靜止 靜止幀算進第一個 subtask 的開頭
充電線插入後手抖/微調 微調算進 subtask 3（plug cable）的結尾，不要切成新 subtask
整集任務失敗（沒完成） 標到最後一個你能辨識的 subtask 結尾即可，後面留空（subtask_index = -1）
💡

標記速度提示：同一個 source repo 的 episodes 動作節奏相近， 標完第一集後後面幾集只需要微調時間點，速度會快很多。 平均每集約 2–5 分鐘可以完成標記。
